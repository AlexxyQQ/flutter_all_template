# flutter_all_template — Mason Brick

A production-ready Flutter starter. Answer 5 questions and get a clean project
with only the modules you actually need.

---

## Prerequisites

```bash
dart pub global activate mason_cli
```

If you plan to use Firebase:
```bash
dart pub global activate flutterfire_cli
```

---

## Usage

### Option A — From this repo (local)

```bash
git clone https://github.com/AlexxyQQ/flutter_all_template
cd flutter_all_template
mason get
mason make flutter_all_template
```

### Option B — From brickhub.dev (once published)

```bash
mason add flutter_all_template
mason make flutter_all_template
```

## Questions asked during generation

| Variable | Type | Description |
|----------|------|-------------|
| `app_name` | string | App name — auto-converted to `snake_case` for the package |
| `app_org` | string | Organization (e.g. `com.example`) |
| `use_firebase` | boolean | Include Firebase (`firebase_core` + `firebase_messaging`) |
| `use_notifications` | boolean | Include push notifications (FCM + `awesome_notifications`) — requires Firebase |
| `use_permissions` | boolean | Include 10 permission screens + `PermissionBloc` + route guard |

---

## What each option includes / excludes

### Firebase (`use_firebase: true`)
- **Adds deps**: `firebase_core`, `firebase_messaging`
- **Keeps**: `lib/config/firebase/firebase_options.dart` (placeholder — run `flutterfire configure`)
- **Without Firebase**: these deps and the directory are removed

### Push Notifications (`use_notifications: true`)
- **Adds deps**: `awesome_notifications`
- **Keeps**: `lib/core/common/domain/services/notification/` (FCM background handler, channels, display logic)
- **Without notifications**: the whole service directory is removed and `bootstrap.dart` / `main.dart` no longer reference it

### Permissions (`use_permissions: true`)
- **Adds deps**: `permission_handler`, `geolocator`, `app_settings`
- **Keeps**: `lib/core/common/features/permission/` (10 permission screens, `PermissionBloc`, `PermissionGuard`)
- **Without permissions**: the feature directory is removed and routes are cleaned from `app_router.dart`

### Always included (core)
BLoC + GetIt/Injectable, auto_route, Dio + interceptors, SecureStorage, Logger,
EasyLocalization, Material 3 theming, AppSettings feature, CRUD BLoC mixins,
Design System widgets, Splash screen.

## Post-generation steps

The hook automatically:
- Renames the package from `flutter_all_template` to your app name in all `.dart` files
- Updates `android/app/build.gradle` with your organization + app name
- Removes disabled feature directories

You still need to run:

```bash
cd <your_app_name>

# If Firebase was selected:
flutterfire configure

# Install deps and regenerate code
flutter pub get
dart run build_runner build --delete-conflicting-outputs

# Run the app
flutter run
```

> **Note**: Auto-generated files (`*.g.dart`, `*.freezed.dart`, `*.gr.dart`,
> `service_locator.config.dart`, `assets.gen.dart`, `locale_keys.g.dart`) are
> intentionally excluded from the brick. They are regenerated by `build_runner`
> after you run the above commands.

## IDE note

Files inside `brick/__brick__/` that contain Mustache syntax (`{{#use_firebase}}`)
will show Dart analyzer errors in the IDE. This is expected — these are template
files, not real Dart code. The root `analysis_options.yaml` already excludes
`brick/**` from analysis.
